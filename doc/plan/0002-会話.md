# 実装計画: 0002-会話

## 概要
現在、1往復のQ&Aのみ対応しているプラグインを、複数回の会話ができるように拡張します。会話履歴を保持し、文脈を考慮した応答を返せるようにします。

## 要件分析

### 現在の実装
- リボンアイコンまたはコマンドから `askAI()` メソッドを実行
- エディタから質問を読み込み (line 51)
- 出力フォーマット: `# Q\n{question}\n\n---\n\n# A\n{AI回答}`
- OpenAI APIをストリーミングで呼び出して回答を取得
- 1回の質問-回答のみ対応

### 要求される変更
1. **会話履歴の解析**: エディタ内の `# Q` と `# A` セクションを解析し、Q-A-Q-A-Qのパターンを抽出
2. **OpenAI APIメッセージ配列の構築**: 会話履歴全体を考慮した messages 配列を作成
3. **回答後の質問プロンプト**: AIの回答の最後に `# Q` の行を追加して次の質問を促す
4. **新しい質問の検出**: 最後の `# Q` セクション以降に書かれた内容を新しい質問として扱う

### 1. Q-A形式のパース機能を追加

#### 新しいインターフェースの定義
```typescript
interface ConversationMessage {
	role: 'user' | 'assistant';
	content: string;
}
```

#### パース関数の追加
```typescript
/**
 * エディタの内容を解析してQ&A履歴と新しい質問を抽出する
 * @param content エディタの全テキスト
 * @returns { history: ConversationMessage[], newQuestion: string }
 */
private parseConversation(content: string): { history: ConversationMessage[], newQuestion: string } {
	const history: ConversationMessage[] = [];
	let newQuestion = '';

	// # Q と # A で分割する正規表現パターン
	// 行頭の # Q または # A でセクションを区切る
	const sections = content.split(/^# (Q|A)$/m);

	// sections[0]: 最初の # Q より前の内容 (通常は空)
	// sections[1]: 'Q' または 'A'
	// sections[2]: その内容
	// sections[3]: 'Q' または 'A'
	// sections[4]: その内容
	// ... の繰り返し

	let currentType: string | null = null;
	let isLastQuestion = false;

	for (let i = 1; i < sections.length; i += 2) {
		const type = sections[i]; // 'Q' または 'A'
		const content = sections[i + 1]?.trim() || '';

		// --- の区切り線を除去
		const cleanContent = content.replace(/^---\s*/m, '').trim();

		if (!cleanContent) {
			// 最後の # Q で内容が空の場合は、これから質問を入力する場所
			if (type === 'Q' && i === sections.length - 1) {
				isLastQuestion = true;
			}
			continue;
		}

		if (type === 'Q') {
			// 最後の Q セクションの場合、これが新しい質問
			if (i >= sections.length - 2) {
				// 最後のセクションまたは最後から2番目(最後が空のAの場合)
				// 次のセクションをチェック
				const nextType = sections[i + 2];
				if (!nextType || nextType === 'Q') {
					// 次のセクションがない、またはQの場合 = まだ回答されていない
					newQuestion = cleanContent;
					isLastQuestion = true;
					continue;
				}
			}
			history.push({ role: 'user', content: cleanContent });
		} else if (type === 'A') {
			// # Q の後の内容が新しい質問として検出されていない場合のみ履歴に追加
			if (!isLastQuestion) {
				history.push({ role: 'assistant', content: cleanContent });
			}
		}
	}

	// エディタの内容全体が # Q や # A を含まない場合
	// 全体を新しい質問として扱う
	if (history.length === 0 && !newQuestion) {
		newQuestion = content.trim();
	}

	return { history, newQuestion };
}
```

**B. エディタフォーマットの更新 (line 60-66の変更)**
```typescript
// 変更前:

// 変更後:
let formattedContent = '';

// 会話履歴が存在する場合は、履歴を保持

	// 既存の会話履歴をフォーマットして保持
	for (let i = 0; i < history.length; i++) {
		const msg = history[i];
		if (msg.role === 'user') {

			formattedContent += `# A\n${msg.content}\n\n---\n\n`;
		}

	// 新しい質問を追加
	formattedContent += `# Q\n${newQuestion}\n\n---\n\n# A\n`;
} else {
	// 会話履歴がない場合は新規フォーマット
	formattedContent = `# Q\n${newQuestion}\n\n---\n\n# A\n`;
}

editor.setValue(formattedContent);
```

## 想定されるエッジケース

### 1. フォーマットが崩れている場合
- **問題**: `# Q` や `# A` が正しく配置されていない
- **対策**: パース時に可能な限り柔軟に対応。最悪の場合は全体を新しい質問として扱う

### 2. 会話履歴が非常に長い場合
- **問題**: OpenAI APIのトークン制限を超える可能性
- **対策**: 現時点では OpenAI API のエラーハンドリングに任せる(既存のtry-catchで対応済み)
- **将来の改善**: トークン数をカウントして古い会話を削除する機能を追加

### 3. ユーザーが手動で # Q や # A を編集した場合
- **問題**: パースロジックが想定外の動作をする可能性
- **対策**: Q-A-Q-A のパターンを前提とし、パターンに合わない場合は最後の # Q 以降を新しい質問として扱う

### 4. 空の # Q セクション
- **問題**: ユーザーが質問を入力せずにコマンドを実行
- **対策**: バリデーションで検出し、エラーメッセージを表示

```
def is_prime(n):
    if n < 2:
        return False
    for i in range(2, int(n**0.5) + 1):
        if n % i == 0:
            return False
    return True

primes = [n for n in range(2, 10001) if is_prime(n)]
```

```
---

# Q
これを高速化するにはどうすればいい?

---

# A
エラトステネスの篩(ふるい)を使うと高速化できます。

def sieve_of_eratosthenes(limit):
    is_prime = [True] * (limit + 1)
    is_prime[0] = is_prime[1] = False

    for i in range(2, int(limit**0.5) + 1):
        if is_prime[i]:
            for j in range(i*i, limit + 1, i):
                is_prime[j] = False

    return [i for i in range(limit + 1) if is_prime[i]]

primes = sieve_of_eratosthenes(10000)
print(primes)

この方法は O(n log log n) の時間計算量で、元の方法よりもはるかに高速です。

---

# Q

```
