# 実装計画: 0003-情報の外出し

## 概要
現在、コード内に直書きされているAIモデル名やシステムプロンプトを、設定画面で設定できるようにします。また、APIキーは機密情報のため、別途安全に保存されるよう設計します。

## 要件分析

### 現在の実装
- **APIキー**: 設定画面で入力可能だが、プラグインのdata.jsonに保存される (line 4, 293-301)
- **AIモデル名**: `gpt-4` がハードコーディングされている (line 198)
- **システムプロンプト**: 未実装(messages配列にsystem roleのメッセージがない)

### 要求される変更
1. **AIモデル名の選択**: ドロップダウンから選択可能に
   - 選択肢: "gpt5.2" (デフォルト), "gpt5.1"
2. **システムプロンプト**: テキストエリアで入力可能に
3. **APIキーの保存方法**: 現在の実装は妥当(Obsidianのプラグインデータとして保存される)

### Obsidianプラグインのデータ保存の仕組み
Obsidianプラグインの設定データは以下の場所に保存されます:
- パス: `<Vault>/.obsidian/plugins/<plugin-id>/data.json`
- 本プラグインの場合: `<Vault>/.obsidian/plugins/obsidian-talk-ai/data.json`
- この仕組みは既に `loadData()` と `saveData()` で実装されている (line 268-274)
- APIキーを含む機密情報は、Obsidian標準の保存方法で保存される
- Vaultごとに独立した設定を持つため、複数のVaultで異なる設定が可能

**注意**:
- Obsidianには専用の「シークレット管理」機能はない
- APIキーはdata.jsonにプレーンテキストで保存される
- ただし、data.jsonはVault内(.obsidianディレクトリ)にあり、通常はVCSから除外される
- これはObsidianプラグインの標準的な実装方法である

2. **今回の実装での対応**:
   - data.jsonへの保存は既存の実装(`loadData()`/`saveData()`)を使用
   - 特別な暗号化は実装しない(Obsidian標準に従う)
   - README.mdにセキュリティ注意事項を記載することを推奨

## 想定されるエッジケース

### 1. システムプロンプトが空の場合
- **問題**: 空のシステムプロンプトをAPIに送信
- **対策**: システムプロンプトが空の場合はmessages配列に追加しない (実装済み)

### 2. 存在しないモデル名が選択された場合
- **問題**: APIエラーが発生
- **対策**: ドロップダウンで選択肢を制限しているため、発生しない
- **追加対策**: APIエラーを既存のtry-catchでハンドリング

### 3. 既存ユーザーの設定マイグレーション
- **問題**: 既存のdata.jsonに新しいフィールドがない
- **対策**: `Object.assign()` で `DEFAULT_SETTINGS` とマージされるため、自動的にデフォルト値が設定される (line 269)

### 4. 非常に長いシステムプロンプト
- **問題**: トークン数を圧迫する可能性
- **対策**: 現時点では制限しない。APIエラーで対応

### 5. モデル名の互換性
- **問題**: "gpt5.2" や "gpt5.1" がOpenAI APIで有効なモデル名か不明
- **対策**:
  - 実装時点では要件通り "gpt5.2" と "gpt5.1" を使用
  - APIエラーが発生した場合、ユーザーに通知される
  - 将来的には実際のOpenAIモデル名に更新することを推奨
  - 参考: 実際のモデル名は "gpt-4", "gpt-4-turbo", "gpt-3.5-turbo" など
