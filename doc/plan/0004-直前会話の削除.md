# 実装計画: 0004-直前会話の削除

## 概要
直前の会話ペア（Q-A）を削除するコマンドを追加します。「Q1->A1->Q2（入力中）」の状態で実行すると「Q1（入力完了状態）」まで戻せる機能です。ユーザーが誤った質問や不要な会話をやり直したい場合に使用します。

## 要件分析

### 現在の実装
- エディタの会話フォーマット: `# Q\n{質問}\n\n---\n\n# A\n{回答}\n\n---\n\n# Q\n`
- 会話履歴の解析機能: `parseConversation()` メソッドで `# Q` と `# A` セクションを分割 (main.ts line 119-180)
- 会話は Q-A-Q-A-Q のパターンで構成される
- `askAI()` コマンドで新しい質問を処理し、回答後に `# Q` を自動追加 (main.ts line 278-281)

### 要求される変更
1. **直前会話の削除コマンド**: 最後の Q-A ペアを削除するコマンドの追加
2. **削除対象の特定**:
   - 最後に完了した Q-A ペアを特定
   - 現在入力中の Q（内容があってもなくても）は削除される
3. **エディタの更新**: 削除後、エディタの内容を更新
4. **エッジケースの処理**:
   - 会話が1往復しかない場合
   - 質問のみで回答がない場合
   - 空のエディタの場合

### 1. `deleteLastConversation()` メソッドの追加

#### 新規メソッドの実装
```typescript
/**
 * 直前の会話ペア(Q-A)を削除し、その前の質問に対する回答を入力できる状態に戻す
 * 例: Q1->A1->Q2 の状態で実行すると Q1->A(空) の状態に戻る
 */
async deleteLastConversation() {
	const activeView = this.app.workspace.getActiveViewOfType(MarkdownView);
	if (!activeView) {
		new Notice('エディタを開いてください');
		return;
	}

	const editor = activeView.editor;
	const content = editor.getValue();

	// エディタが空の場合
	if (!content || content.trim() === '') {
		new Notice('削除する会話がありません');
		return;
	}

	// 会話履歴を解析
	const { history, newQuestion } = this.parseConversation(content);

	// 削除可能な会話がない場合
	// - history が空 = 初回の質問のみ
	// - history が1つのみ = Q のみで A がない状態
	if (history.length === 0) {
		new Notice('削除する会話がありません（最初の質問のみです）');
		return;
	}

	// history の最後のペアを削除
	// history は [user, assistant, user, assistant, ...] の順で構成される
	// 最後が assistant の場合: 最後の Q-A ペアを削除
	// 最後が user の場合: 最後の Q のみ削除（回答がまだない）

	let newHistory: ConversationMessage[] = [];

	if (history.length >= 2) {
		// 最後の Q-A ペア（または Q のみ）を削除
		// history[length-2] が user、history[length-1] が assistant の場合は両方削除
		// history[length-1] が user の場合は最後の user のみ削除

		const lastMessage = history[history.length - 1];

		if (lastMessage.role === 'assistant') {
			// Q-A のペアが完了している場合
			// 最後の2つ（user + assistant）を削除
			newHistory = history.slice(0, -2);
		} else {
			// 最後が user の場合（回答がない状態）
			// 最後の user を削除
			newHistory = history.slice(0, -1);
		}
	} else if (history.length === 1) {
		// history が1つのみの場合
		// これは最初の Q に対する A がある状態、または Q のみの状態
		if (history[0].role === 'user') {
			// Q のみの場合 = 削除不可
			new Notice('削除する会話がありません（最初の質問のみです）');
			return;
		} else {
			// あり得ないケースだが、念のため
			newHistory = [];
		}
	}

	// 新しいエディタの内容を構築
	let newContent = '';

	if (newHistory.length === 0) {
		// すべての会話を削除した場合 = 空のエディタ
		newContent = '';
	} else {
		// 残った会話履歴を再構築
		for (let i = 0; i < newHistory.length; i++) {
			const msg = newHistory[i];
			if (msg.role === 'user') {
				newContent += `# Q\n${msg.content}\n\n---\n\n`;
			} else {
				newContent += `# A\n${msg.content}\n\n---\n\n`;
			}
		}

		// 最後に空の # A を追加（次の回答を入力できる状態）
		newContent += `# A\n`;
	}

	// エディタの内容を更新
	editor.setValue(newContent);

	// カーソルを最後の # A の後ろに移動
	const lastLine = editor.lastLine();
	editor.setCursor({ line: lastLine, ch: editor.getLine(lastLine).length });

	new Notice('直前の会話を削除しました');
}
```

## 削除ロジックの詳細

### `parseConversation()` の返り値の構造

#### 例1: Q1->A1->Q2(空) の場合
```javascript
{
  history: [
    { role: 'user', content: 'Q1の内容' },
    { role: 'assistant', content: 'A1の内容' }
  ],
  newQuestion: ''
}
```
→ 削除後: `history.slice(0, -2)` = `[]`（空）

#### 例2: Q1->A1->Q2(入力済み) の場合
```javascript
{
  history: [
    { role: 'user', content: 'Q1の内容' },
    { role: 'assistant', content: 'A1の内容' }
  ],
  newQuestion: 'Q2の内容'
}
```
→ 削除後: `history.slice(0, -2)` = `[]`（空）

#### 例3: Q1->A1->Q2->A2->Q3 の場合
```javascript
{
  history: [
    { role: 'user', content: 'Q1の内容' },
    { role: 'assistant', content: 'A1の内容' },
    { role: 'user', content: 'Q2の内容' },
    { role: 'assistant', content: 'A2の内容' }
  ],
  newQuestion: ''
}
```
→ 削除後: `history.slice(0, -2)` =
```javascript
[
  { role: 'user', content: 'Q1の内容' },
  { role: 'assistant', content: 'A1の内容' }
]
```

## 想定されるエッジケース

### 1. 空のエディタの場合
- **問題**: 削除する内容がない
- **対策**: 「削除する会話がありません」とエラーメッセージを表示

### 2. 最初の質問のみの場合
- **問題**: Q1のみで回答がない状態
- **対策**: 「削除する会話がありません（最初の質問のみです）」とエラーメッセージを表示

### 3. フォーマットが崩れている場合
- **問題**: `# Q` や `# A` が正しく配置されていない
- **対策**: `parseConversation()` が既に柔軟にパースしているため、そのまま利用

### 4. 削除後に空になる場合
- **問題**: 最後の Q-A ペアを削除した結果、エディタが空になる
- **対策**: 空のエディタにして通知を表示

### 5. ユーザーが手動でフォーマットを編集した場合
- **問題**: 想定外のフォーマット
- **対策**: `parseConversation()` の解析結果に基づいて動作

### 6. 削除コマンドを連続して実行した場合
- **問題**: 複数回実行すると会話がどんどん削除される
- **対策**: 各実行時に `parseConversation()` で現在の状態を判定し、削除可能かチェック
